For now, we exit the VM and check out the hardening.patch since that should be giving us a detailed review of what they did to harden the OS.

Main things here are:

wrmsr stands for write model specific register(wrmsr) and it writes some value to a register. The MSR_FMASK register is the syscall register which is the toggle for users to run CPU features. The hardening being done in this line is that the changed 0x200 to 0x600. In this case, the actual hexadecimal isn’t the key here, it’s the bits. These bits are CPU flags and it controls what the users can do. The binary for 0x200 is 0010 0000 0000. The 9th bit is flipped. That bit controls the interrupts on syscall, which prohibits users to interrupt while the kernel is still setting up the registers and everything. Because the whole thing here is that the user needs to syscall and use some CPU features and so the kernel needs to set up some stuf first and the 9th bit just prevents that from happening. By chaning the 0x200 to 0x600, it is changing the bits to 0110 0000 0000. Ok, so it’s also flipping the 10th bit, which as the comment suggests, is disabling DF or the Direction Flag. This only changes a String Instructions such as memcpy or memset. The direction flag changes the order it is being copied or just how the instruction is being carried out. Normally, it would go from front to back. The DF simiply chages that and does the instructions back to front. The end results are still the same and everything, but it’s hardened becaues nobody would know if your stuff is copying from front to back or the other way.

The next one here changes the above. The changes are in the - and +. Again, since it’s CPU, it’s just changing flags and whatnot. In the original one, only the 9th and 10th bit are set to 1. The comment above explains it well. Those two bits are:

Bit 9 SFXSR (Operating System Fast FXSAVE/FXRSTOR Support):
This bit is part of the two bits that make up SSE. The scond bit is the 10th bit which I will explain later on. This bit adds support for XMM0~XMM15 registers. Those registers are used for complex math such as for cryptography, graphics, or 3D rendering. Another common usage is when a process is being closed and the cpu must really quickly save the state and data it had on so that it can be opened later on in the same state. Basically, it just enables these registers for sum big math.

Bit 10 OSXMMEXCPT(Operating System XMM Unmasked Exception Support):
This is features for those XMM registers(0~15). This cpu flag is to ensure no problems when an error is made. Without this, if there was an errror happening in the cpu process, it’d just crash, but this flag gives the control back to the kernel and makes it able to shut it down safely. The whole thing is KIND OF in the name it being OS XMM register unamsked(not hidden error) support for exceptions.
