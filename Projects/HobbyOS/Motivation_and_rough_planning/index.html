Motivation and Rough Planning

Lucas Huang
05-24-2026 (Updated: 05-28-2026)
Projects › HobbyOS
distro, hobbyos, lfs, linux, planning, project

Initial Remarks

Even though this is a bit off topic, this is pretty much my first actual post on this blog. I know the KalmarCTF one was before this but I made that thing a while ago so this is my first post made FOR this blog. It’s also the first project I’ll be fully documenting on this blog. Also, sorry if this post isn’t very structured. I’m still new to writing these blog posts. I’m sure I’ll get better at it as I make more posts.

Motivations

Alright guys, so I’ve known about LFS(Linux From Scratch) and BLFS(Beyong LFS) for quite a bit and since I’m also planning this summer to configure and move to Gentoo as my daily drive on my main laptop, I was just like hmmmmm, what if I also just made a fun LFS build just to learn a bit more about how linux works. Later on, I realized that I could make it a bit polished with BLFS and swaywm to it and rice it a little to make it a full hobby distro.

After brainstorming it a bit, I thought that it would be cool that since I’m interested in cybersecurity and specifically pwn and rev, to just make a minimal linux distro with some tools and stuff already made. I’m sure somebody else has already done this type of stuff, but it’s really just for my understanding of linux and also to have some fun with it.

That is literally the entire reason why I’m doing this lmaoooo💀💀. Maybe I’ll think of some more grand reason later on when I finish the distro and make a page for it.

Rough Plan

I already finished making the html pages for LFS and BLFS(holy shit it is long). As you can tell from the top, this post is made on May 24 and I really only plan on actually following the LFS and stuff in the Summer. I also got exams as well, so I’ll only be able to actually do stuff once school ends. From now till then, I’ll just try to read over the LFS to get a good idea of what it is doing.

Later, I’ll configure the LFS first by following the guide. As of now, I still don’t know jack about how it’s going to go so I can’t really say any specific thing but I’ll just follow the LFS to get a base CLI distro then I’ll follow the BLFS to get a swaywm thing on top. I honestly can NOT bother with GNOME or KDE stuff. I have a feeling that the apps and dependencies that come with those would kill me before I make a full distro.

After the initial configuration, I’ll make that into an iso to understand how that process works.

Then, I’ll go back and this time, I’ll diverge from the main LFS guide and install OPENRC as my init. Probably with some help from some AI and random geniuses in Discord groups I’m in.

Vision

So the vision is to obviously have a minimal distro with a minimal riced swaymd and a bunch of security features in the distro turned on. The init system will be OPENRC in the end and it’ll also come with some pwn and rev tools such as gef or ghidra or binary ninja.

Final Stuff

I really have no idea how big of a project I’m getting myself into but I’m pretty exicted to get started. Yuuhhhhh, in these projects, I’ll also try to give myself some listed homework at the bottom to just keep myself on track.
